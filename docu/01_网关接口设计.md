# 算法 Agent 网关与返回数据结构设计

## 一、整体架构

```
┌─────────────────┐
│  竞赛平台浏览器  │
│  (fetch 调用)    │
└────────┬────────┘
         │ HTTP/JSON
         ▼
┌─────────────────┐
│  Agent 网关      │ ← 选手本机或同内网，Base URL 如 http://127.0.0.1:8080
│  (FastAPI 等)    │
└────────┬────────┘
         │ 路由分发
    ┌────┼────┐
    ▼    ▼    ▼
 classify detect ocr   ← 三个算法分支
```

---

## 二、必须实现的两个端点

| 端点 | 方法 | 作用 |
|------|------|------|
| `/health` | GET | 健康检查，平台据此判断服务是否在线 |
| `/infer` | POST | 推理接口，所有题型共用此入口 |

**部署约束：**
- Base URL 必须为根地址，如 `http://127.0.0.1:8080`，平台自行拼接 `/health` 和 `/infer`
- 必须返回合规 CORS 头（含 OPTIONS 预检）
- 字符编码 UTF-8，Content-Type: application/json

---

## 三、GET /health

### 请求
```
GET {agent_base_url}/health
```

### 响应（必须返回）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | string | 是 | 必须为 `"ok"` 表示可接受评测 |
| supported_tasks | string[] | 是 | 已实现的 task_type 列表，须为本队要打的题型子集 |

**可选字段：**
- `service`: 服务名，如 `"contestant-algo-test-service"`
- `version`: 版本号
- `bridge_mode`: 如 `"mock-local"`

### 响应示例
```json
{
  "status": "ok",
  "supported_tasks": ["classify", "ocr", "detect"]
}
```

## 四、POST /infer — 通用信封格式

### 4.1 请求体（所有题型共用）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| request_id | string | 是 | 平台生成，全局唯一，响应必须原样回显 |
| session_id | string | 是 | 平台生成，响应可省略 |
| task_type | string | 是 | `"classify"` / `"ocr"` / `"detect"`（小写） |
| image | object | 是 | 必须含 `format` 和 `data`，常见 `{"format": "url", "data": "https://..."}` |
| meta | object | 否 | 判分上下文，可含 difficulty、sample_index、infer_T_max_ms 等 |

### 4.2 响应体（所有题型共用）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| request_id | string | 是 | 必须与请求完全一致 |
| task_type | string | 是 | 必须与请求一致（平台大小写不敏感比对） |
| ok | boolean | 是 | true=进入判分，false=本次失败（仍需返回合法JSON） |
| result | object/null | 是 | 成功时放算法结果，失败时为 null |
| message | string | 是 | 失败原因；成功时必须为 `""`（空字符串） |
| elapsed_ms | number | 是 | 整数毫秒，含处理耗时 |

**响应字段顺序建议：** request_id → task_type → ok → result → elapsed_ms → message

### 4.3 失败响应示例
```json
{
  "request_id": "eval-1205-1-1",
  "task_type": "ocr",
  "ok": false,
  "result": null,
  "elapsed_ms": 128,
  "message": "推理超时"
}
```

## 五、task_type: "classify" — 图像分类

### 请求示例
```json
{
  "request_id": "eval-1205-1-1",
  "session_id": "team-7",
  "task_type": "classify",
  "image": {
    "format": "url",
    "data": "https://platform.example.com/files/signed/def456.jpg"
  },
  "meta": {
    "difficulty": "L1",
    "sample_index": 1,
    "sample_k": 1,
    "infer_T_max_ms": 15000,
    "class_names": ["正常", "告警", "离线"]
  }
}
```

### result 内字段
| 字段 | 必填 | 说明 |
|------|------|------|
| label | 是 | 单标签字符串，必须落在当次 `meta.class_names` 内（大小写不敏感） |

### 响应示例
```json
{
  "request_id": "eval-1205-1-1",
  "task_type": "classify",
  "ok": true,
  "result": {
    "label": "正常"
  },
  "elapsed_ms": 42,
  "message": ""
}
```

### 固定候选类别（平台下发）
办公室、公园、街道、商场、厨房、卧室、图书馆、体育馆（共8类）

## 六、task_type: "detect" — 目标检测

### 请求示例
```json
{
  "request_id": "eval-1205-1-1",
  "session_id": "team-7",
  "task_type": "detect",
  "image": {
    "format": "url",
    "data": "https://platform.example.com/files/signed/ghi789.jpg"
  },
  "meta": {
    "difficulty": "L2",
    "sample_index": 1,
    "sample_k": 1,
    "infer_T_max_ms": 15000,
    "coord_mode": "pixel",
    "scoring": "center_in_box",
    "class_names": ["人", "汽车", "自行车", "手机", "水杯", "笔记本电脑", "台灯", "沙发", "狗"],
    "image_width": 640,
    "image_height": 480
  }
}
```

### result 内结构
```json
{
  "targets": [
    {
      "label": "笔记本电脑",
      "cx": 980.5,
      "cy": 769.0,
      "score": 0.942
    }
  ]
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| label | 是 | 类别字符串，须在 `meta.class_names` 内 |
| cx | 是 | 浮点数，目标中心点 x 坐标（**像素**） |
| cy | 是 | 浮点数，目标中心点 y 坐标（**像素**） |
| score | 否 | 置信度，平台不强制校验 |

### 响应示例
```json
{
  "request_id": "eval-1205-1-1",
  "task_type": "detect",
  "ok": true,
  "result": {
    "targets": [
      { "label": "笔记本电脑", "cx": 980.5, "cy": 769.0 },
      { "label": "手机", "cx": 376.0, "cy": 994.5 }
    ]
  },
  "elapsed_ms": 118,
  "message": ""
}
```

### 固定候选类别（平台下发，共9类）
人、汽车、自行车、手机、水杯、笔记本电脑、台灯、沙发、狗

### 坐标系约束
- **必须为像素坐标**，`meta.coord_mode` 固定为 `"pixel"`
- 原点为题图左上角，x 向右、y 向下
- **禁止**使用 0~1 归一化坐标

### 判分逻辑
- 不按 IoU 算框重叠率
- 只看：①类别是否一致 ②中心点 (cx,cy) 是否落入对应评委参考框
- 得分 = 命中目标数 / 标准目标总数 × 100

|------|------|------|

```json

  "task_type": "ocr",
  "ok": true,
  "result": {
    "text": "阀位12.8%"
  },
  "elapsed_ms": 85,
  "message": ""
}
```

## 八、错误码与异常处理

| 场景 | 处理方式 |
|------|----------|
| 不支持的 task_type | 返回 `ok: false`，message 说明不支持 |
| 超时 | 单次推理须在 `meta.infer_T_max_ms` 内完成 |
| HTTP ≥400 或非 JSON | 当次抽样失败，整题按平台规则失败 |
| CORS 缺失 | 浏览器无法发请求 |

---

## 九、交付自检清单

- [ ] `GET /health` 返回 200 + JSON，`status === "ok"`
- [ ] `supported_tasks` 包含本队参赛所需的全部 task_type
- [ ] `POST /infer` 三种题型均能返回完整信封且 result 符合结构
- [ ] `request_id` / `task_type` 响应与请求一致
- [ ] classify: `result.label` 属于当次 `meta.class_names`
- [ ] detect: 返回像素坐标 `cx`/`cy`，非归一化坐标
- [ ] ocr: `result.text` 存在且为字符串
- [ ] 跨域、超时已在真实浏览器环境下验证通过

## 十、快速实现参考（Python FastAPI）

```python
from fastapi import FastAPI, Request
from fastapi.middleware.cors import CORSMiddleware
import time

app = FastAPI()

# CORS 配置
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/health")
async def health():
    return {
        "status": "ok",
        "supported_tasks": ["classify", "ocr", "detect"]
    }

@app.post("/infer")
async def infer(request: Request):
    start = time.perf_counter()
    body = await request.json()
    
    request_id = body["request_id"]
    task_type = body["task_type"].strip().lower()
    
    # 根据 task_type 分发到对应算法
    if task_type == "classify":
        result = await do_classify(body)
    elif task_type == "detect":
        result = await do_detect(body)
    elif task_type == "ocr":
        result = await do_ocr(body)
    else:
        return {
            "request_id": request_id,
            "task_type": task_type,
            "ok": False,
            "result": None,
            "elapsed_ms": int((time.perf_counter() - start) * 1000),
            "message": f"不支持的 task_type: {task_type}"
        }
    
    return {
        "request_id": request_id,
        "task_type": task_type,
        "ok": True,
        "result": result,
        "elapsed_ms": int((time.perf_counter() - start) * 1000),
        "message": ""
    }
```
