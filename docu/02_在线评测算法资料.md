# 在线评测算法资料

## 一、评测总体结构

### 1.1 任务类型
平台支持三种视觉任务：
- **classify** — 图像分类（工况/场景分类）
- **detect** — 目标检测（工业目标定位）
- **ocr** — 文字识别（表计/铭牌读数）

### 1.2 难度档位
所有题型统一分三档：

| 档位 | 难度 | 能力侧重 |
|------|------|----------|
| L1 | 易 | 题面规整，对比度高，透视/遮挡少；可用传统图像处理+轻量模型 |
| L2 | 中 | 光照、视角、尺度变化，有一定干扰；需数据增强+经典小模型 |
| L3 | 难 | 细粒度、密集、弱对比、类间易混；需深度学习，强调鲁棒性 |

### 1.3 槽位与计分

共 3类 × 3档 = **9个槽位**，每个槽位可多次抽图重试（同槽位最多新增2次），取最高分。

| 题型 | L1 满分 | L2 满分 | L3 满分 | 小计 |
|------|---------|---------|---------|------|
| classify | 4.8 | 6.0 | 7.2 | **18 分** |
| ocr | 4.8 | 6.0 | 7.2 | **18 分** |
| detect | 6.0 | 7.8 | 10.2 | **24 分** |
| **合计** | | | | **60 分** |

**单槽得分 = 原始得分 / 100 × 该槽满分**

## 二、各题型详细说明

### 2.1 图像分类 (classify)

#### 固定类别（8类）
| 序号 | 场景类别 |
|------|----------|
| 1 | 办公室 |
| 2 | 公园 |
| 3 | 街道 |
| 4 | 商场 |
| 5 | 厨房 |
| 6 | 卧室 |
| 7 | 图书馆 |
| 8 | 体育馆 |

#### 各难度命题特点
| 难度 | 命题主轴 |
|------|----------|
| L1 | 3~5类，类间差异大 |
| L2 | 更多类、细粒度（相似外观子类） |
| L3 | 多标签或含干扰类；需拒识或多头输出策略 |

#### 判分逻辑
- 返回 `result.label` 与标准类别一致 → 原始得分 100
- 否则 → 原始得分 0
- 标签不在合法类别中也按错误处理

#### 返回格式
```json
{
  "result": {
    "label": "办公室"
  }
}
```

---

### 2.2 目标检测 (detect)

#### 固定类别（9类）
| 序号 | 检测类别 |
|------|----------|
| 1 | 人 |
| 2 | 汽车 |
| 3 | 自行车 |
| 4 | 手机 |
| 5 | 水杯 |
| 6 | 笔记本电脑 |
| 7 | 台灯 |
| 8 | 沙发 |
| 9 | 狗 |

#### 各难度命题特点
| 难度 | 命题主轴 |
|------|----------|
| L1 | 单类或少实例，朝向较正，框尺度适中 |
| L2 | 多类多实例，尺度变化与部分遮挡，含小目标 |
| L3 | 密集、弱对比、类间易混，需控制误检与漏检 |

#### 坐标系（固定）
- **必须为像素坐标**
- `meta.coord_mode` 恒为 `"pixel"`
- 原点为题图左上角，x 向右、y 向下
- **禁止**使用 0~1 归一化坐标

#### 判分逻辑（重点！）
**不按 IoU 算框重叠率**，只看两件事：
1. **类别写对**：返回的 label 必须与标准目标的类别一致
2. **中心点落入框内**：返回的 (cx, cy) 必须落在对应的评委参考框内

得分计算：
```
原始得分 = 命中的标准目标数 / 标准目标总数 × 100
最终槽位得分 = 原始得分 / 100 × 当前题型难度满分
```

**示例：** 一张图有10个标准目标，你命中6个（类别对+中心点在框内），原始得分=60分。若该槽满分10.2，最终得分 = 60/100 × 10.2 = 6.12分

#### 推荐返回格式
优先返回 `targets`，直接给中心点：
```json
{
  "result": {
    "targets": [
      { "label": "笔记本电脑", "cx": 980.5, "cy": 769.0 },
      { "label": "手机", "cx": 376.0, "cy": 994.5 }
    ]
  }
}
```

也可返回 `boxes[].xyxy`，平台会自动取框中心点。

### 2.3 文字识别 (ocr)

#### 各难度命题特点
| 难度 | 命题主轴 |
|------|----------|
| L1 | 单行、规整印刷体或数码管类读数，高对比、少粘连 |
| L2 | 多行、轻透视或阴影，需分行或整体拼接策略 |
| L3 | 强反光、污渍、小字密集，宜"检测文本行+识别"或端到端鲁棒模型 |

#### 判分逻辑
- 平台按 `meta.normalize_rules` 对标准文本和模型输出做同样规范化
- 完全一致 → 原始得分 100
- 否则 → 原始得分 0

#### normalize_rules 常见规则
| 字段 | 说明 |
|------|------|
| trim_space | 是否去除首尾空格 |
| case_insensitive | 英文是否忽略大小写 |

#### 返回格式
```json
{
  "result": {
    "text": "阀位12.8%"
  }
}
```

## 三、请求 meta 字段说明

平台在 `/infer` 请求的 `meta` 中下发关键信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| difficulty | string | 难度档位："L1"/"L2"/"L3" |
| sample_index | int | 当前是第几次抽样 |
| sample_k | int | 该槽位总共可抽样次数 |
| infer_T_max_ms | int | 本次推理最大允许耗时（毫秒） |
| class_names | string[] | 合法类别列表（classify/detect 使用） |
| coord_mode | string | 坐标模式，固定为 "pixel" |
| language_hint | string | OCR 语言提示，如 "zh" |
| normalize_rules | object | OCR 规范化规则 |
| image_width | int | 题图宽度参考（像素） |
| image_height | int | 题图高度参考（像素） |

## 四、计分流程总结

```
选手提交 → 平台抽图 → 调用 /infer → 返回 result → 判分器计算原始得分(0~100)
    ↓
原始得分 / 100 × 槽位满分 = 单槽得分
    ↓
9个槽位得分求和 = 在线评测总分（最高60分）
    ↓
同一槽位多次成功，取最高分
```

## 七、常见问题

**Q1: detect 的框需要多精确？**
A: 不需要精确框，只要中心点落入标准框即可。建议直接输出中心点而非完整框。

**Q2: 可以同一个目标返回多次吗？**
A: 不可以。每个标准目标最多匹配一个预测目标，每个预测目标也最多用一次。

**Q3: ocr 大小写敏感吗？**
A: 取决于 `meta.normalize_rules.case_insensitive`，先对预测串做同一套规范化再比对。

**Q4: classify 返回的标签可以是英文吗？**
A: 不可以。必须返回 `meta.class_names` 中的中文标签。

**Q5: 超时了怎么办？**
A: 单次推理须在 `meta.infer_T_max_ms` 内完成，超时则该次抽样失败。
